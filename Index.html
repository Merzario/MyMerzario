API-referentie · v1

Volg zendingen met
elke referentie die je hebt.

Eén alleen-lezen endpoint dat de actuele status, mijlpalen en route van een zending of container teruggeeft — op basis van het containernummer, de house bill, de orderreferentie of het shipment-ID dat je al hebt. Geen login, geen setup.

Overzicht

Deze API geeft je programmatische toegang tot dezelfde tracking-informatie die je in ons publieke tracking-portal ziet, afkomstig uit ons WiseTech Global CargoWise-systeem. Gebruik het om zendingstatus in je eigen systemen te halen — een portal, dashboard, ERP of een geautomatiseerde statusmail.

Wat het teruggeeft: actuele status, de historie van mijlpalen, geschatte en werkelijke vertrek-/aankomsttijden, voortgang en de routedelen van een zending. Wat het niet doet: het kan niets wijzigen en toont geen commerciële gegevens zoals tarieven, kosten, facturen of partijgegevens.

Toegang & authenticatie

Er is geen API-sleutel, token of login nodig. Het endpoint herleidt elke geldige referentie die je meestuurt. In de praktijk ís het referentienummer de sleutel: iedereen die een geldig container- of B/L-nummer kent, kan de tracking-status opvragen — net als op het publieke portal.

Goed om te wetenOmdat er geen credentials zijn, hoef je niets te roteren of veilig op te slaan. Houd simpelweg je lijst met referentienummers bij en roep het endpoint aan.

Werken met de data

Een paar veldconventies om in je parser af te vangen:

Checklist voor robuust verwerken
  • Splits op entityType, niet op de referentie die je stuurde — de data-vorm verschilt.
  • WEIGHT is een string met eenheid ("10386 KG"). Haal het getal eraf als je wilt rekenen.
  • TOTALPACKAGES is een string. Parse naar geheel getal waar nodig.
  • CONTAINERTYPE kan rechts met spaties zijn aangevuld ("40HC   "). Trim het.
  • VESSELCODE bevat de scheepsnaam, ondanks de veldnaam.
  • Gebruik voor tijd bij voorkeur eventTimeUtc; behandel velden zonder offset als lokale weergavetijd.
  • Ontbrekende referenties vervallen, dus koppel resultaten terug op referentie, niet op arraypositie.
  • progress wisselt licht tussen identieke calls (tijdgebaseerd). Gebruik het niet als cache-sleutel.
Aanvragen versturenDe enige header die je nodig hebt is Content-Type: application/json. Trace- of proxy-headers die je in opgevangen verkeer ziet (Datadog, traceparent, X-Forwarded-For, enz.) worden door clients toegevoegd en zijn niet vereist.

Limieten & stabiliteit

Lees dit voordat je gaat bouwen
  • Dit endpoint voedt ons tracking-portal en wordt as-is aangeboden, zonder gepubliceerde SLA. Veldnamen en gedrag kunnen zonder aankondiging wijzigen — verwerk defensief zodat een onverwacht veld je nooit breekt.
  • De host draait achter Cloudflare, dat bij grote automatische pieken kan rate-limiten of een challenge geven. Houd het volume redelijk, bundel referenties in één call en cache responses een paar minuten in plaats van strak te pollen. Bouw exponentiële back-off in bij 429/5xx.
  • Gebruik exact de getoonde basis-URL; de host is specifiek voor onze omgeving.
  • Behandel referentienummers als laag-gevoelig, maar wel privé voor de partijen op de zending.

Vragen over de werking of de data? Mail ons team op mymerzario@merzario.nl. Voor algemene support kun je terecht bij support@merzario.nl.

Track shipments by

A single, read-only endpoint that returns the live status, milestones and route of a shipment or container — using the container number, house bill, order reference or shipment ID you already have. No login, no setup.

Overview

This API gives you programmatic access to the same tracking information shown in our public tracking portal, drawn from our WiseTech Global CargoWise system. Use it to pull shipment status into your own systems — a portal, a dashboard, an ERP, or an automated status email.

What it returns: current status, the milestone history, estimated and actual departure/arrival times, journey progress, and the route legs of a shipment. What it does not do: it cannot change anything, and it does not expose commercial data such as rates, charges, invoices or party details.

Access & authentication

No API key, token or login is required. The endpoint resolves any valid reference you send it. In practice the reference number is the access key: anyone who knows a valid container or B/L number can read its tracking status, exactly as they can on the public web portal.

Good to knowBecause there are no credentials, there is nothing to rotate or store securely. Just keep your list of reference numbers and call the endpoint.

Working with the data

A few field conventions to handle in your parser:

Defensive parsing checklist
  • Branch on entityType, not on the reference you sent — the data shape differs.
  • WEIGHT is a string with a unit ("10386 KG"). Split off the number if you need maths.
  • TOTALPACKAGES is a string. Parse to integer where needed.
  • CONTAINERTYPE can be right-padded with spaces ("40HC   "). Trim it.
  • VESSELCODE contains the vessel name, despite the field name.
  • For time, prefer eventTimeUtc; treat the offset-less fields as display-only local times.
  • Missing references are dropped, so match results back to your input by reference rather than by array position.
  • progress drifts slightly between identical calls (it is time-based). Don’t use it as a cache key.
Sending requestsThe only header you need is Content-Type: application/json. Any tracing or proxy headers you see in captured traffic (Datadog, traceparent, X-Forwarded-For, etc.) are added by individual clients and are not required.

Limits & stability

Please read before you build
  • This endpoint powers our tracking portal and is provided as-is, with no published SLA. Field names and behaviour may change without notice — wrap parsing so an unexpected field never breaks you.
  • The host sits behind Cloudflare, which may rate-limit or challenge bursty automated traffic. Keep request volume reasonable, batch references into single calls, and cache responses for a few minutes rather than polling tightly. Back off on 429/5xx with exponential retry.
  • Use the exact base URL shown; the host is specific to our environment.
  • Treat reference numbers as low-sensitivity but still private to the parties on the shipment.
